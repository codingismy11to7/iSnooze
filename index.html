iSnooze

About iSnooze

iSnooze sits in your system tray and lets you schedule times for iTunes to start playing. Spacebar activated snooze, configurable snooze time, gradual volume increasing, multiple alarm schedules, playlist selection, temporary pausing when you reach your computer to give you time to shut it off...why I daresay iSnooze has it all!

Release 1.3: SetupiSnooze-1.3.exe "Oops"
Features
  • You can now disable alarms from the configuration screen. This is useful, for instance, if you have a temporary schedule change one day, but don't want to delete your normal scheduled alarms.
  • The system tray icon now turns red if the alarm is not set to go off, for any reason: no alarms are set, alarms are disabled, all alarms are set to disabled, no alarms have any days set...
  • Changed the volume increaser to start from 30% instead of 0%. iTunes is practically silent for that whole range anyway.
  • The most important change...iTunesAlarm has been renamed to iSnooze. When samyi.org suggested I make this program, I was said to myself "hmmm, ok, sounds like a good idea" without even bothering to check if it already existed. Well, various programs like iSnooze exist, but I still think mine is the best, so I will continue to maintain it, and as always, it is Open Source. I had to change the name because "iTunesAlarm" already exists. My apologies to Bill Zeller, I didn't mean to create any confusion. This time around, I did search Google for "iSnooze" and I think it's OK. Thanks to samyi.org for the name, and thanks to everybody else who helped. It was almost named "Dream Destroyer 3000" thanks to Emily...but that's just too harsh:) BTW, great minds must think alike, or something, because iSnooze & Bill Zeller's O.G. iTunesAlarm are alike in almost every way, it was crazy. Same installer, most of the same features, same name, same registry location...we even both considered deleting the registry key on Uninstall (I changed my mind, he left it in), which created some confusion when I uninstalled his iTunesAlarm for the first time. That'll teach me to not neglect using a VendorName under HKCU\Software...
